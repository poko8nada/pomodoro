# pomodoro

ポモドーロタイマーを中心に組み立てている、小さな実験用の Web アプリ。

## 目的

- 小さなことを始めるのに躊躇してしまう自分の改善のため
- 手戻りや間違いを嫌う精神構造の改善のため、短い実験として Todo を捉え直す試み

## 現在の実装

- 現在は **タイマー先行**。トップページにはポモドーロタイマー UI がある
- 集中時間・休憩時間は **5 分刻み** で調整できる
- 集中時間 + 休憩時間の合計は **60 分以内**
- サイクル数は **1〜5 回** から選べる
- 開始 / 一時停止 / 再開 / リセットができる
- 現在フェーズ（focus / break）と、残りサイクル数 / 全サイクル数を表示する
- Todo 一覧、Todo 選択、タイマーとの結線は **まだ未実装**

## 次にやること

- Todo 最小 CRUD
- Todo とタイマーの結線
- 画面を「上: タイマー」「下: Todo」に寄せたレイアウト整理

## データの置き場

- 現時点のタイマー設定・進行状態は **メモリ上のみ**
- 永続化が必要になった段階では、まずブラウザの `localStorage` を候補にする

## タイマー実装

- 締切ベース: 終了時刻を `Date.now() + 残りミリ秒` のように持ち、表示は常に `max(0, endTime - Date.now())` から算出する。裏タブで `setInterval` が間引かれても、論理上の残り時間は破綻しない
- 補助: `visibilitychange` で `document.visibilityState === 'visible'` のときに、上記の式で一度再計算して UI を更新する（表に戻った直後の表示ズレを減らす）
- タイマー開始時に、その時点の設定をセッション側へスナップショットする。開始後に設定 UI を触っても、進行中のセッション長は変わらない
- セッション状態は `idle / running / paused` と `focus / break` を軸に管理する
- PWA / プッシュ通知: タイマー精度の問題とは別軸のため、いったん検討しない

## 技術スタック

- Honox（ファイルルーティングは使えるが、当面はトップのみでよい）
- Tailwind CSS v4（`@tailwindcss/vite`）
- daisyUI（Tailwind のプラグイン。コンポーネント用クラスとテーマ。`app/style.css` で `@plugin 'daisyui'`）
  - themeは `nord` 。ダークモードは基本なし。
- パッケージ管理: `pnpm`（詳細は `package.json`）
